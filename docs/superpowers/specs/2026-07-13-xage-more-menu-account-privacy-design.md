# XAGE 更多菜单、账号安全与隐私页面设计

日期：2026-07-13
状态：已确认

## 目标

在不改变 `XAgeMainView` 主页面结构的前提下，完成以下改进：

1. 从 `XAgeMoreMenu` 进入“报告、日常、就医、画像”后，子页面返回时仅关闭子页面，继续停留在更多菜单。
2. 在账号管理中新增“账号与安全”，集中展示脱敏手机号、修改密码和注销账号入口。
3. 在关于区域新增本地“隐私政策”和“权限申请与使用情况说明”页面。

## 现状与问题

`XAgeMoreMenu` 通过 `fullScreenCover(item:)` 展示四个资料分类。子页面左上角返回会先将 `presentedCategory` 置空，但 cover 的 `onDismiss` 随后调用 `closeMenuAfterCategoryDetail()`，后者又执行 `XAgeMoreMenu.onClose()`，因此更多菜单也被关闭。

项目已经具备以下可复用能力：

- `UserInfo.phone` 以及 `/api/users/me` 用户信息接口。
- `ChangePasswordViewModel`、`ChangePasswordSheet` 和 `/api/auth/password/change` 修改密码接口。
- `XAgeAccountViewModel.deleteAccountOnServer()` 和 `DELETE /api/users/me` 注销接口。
- 现有不可逆操作确认页 `XAgeDeleteAccountSheet`。
- `docs/privacy.html` 中的正式隐私政策内容。
- `Info.plist` 和 `PrivacyInfo.xcprivacy` 中的权限用途与隐私声明。

## 方案选择

采用局部状态配合独立 `fullScreenCover` 的方案，延续 `XAgeMoreMenu` 当前的呈现结构。

不在本次任务中将整个更多菜单重构为 `NavigationStack` 或全局路由器。那些方案虽然能统一导航状态，但会同时影响家庭模式、个人信息、用药管理和既有弹层，改动范围与本次需求不成比例。

## 页面与导航设计

### 资料分类返回

- 保留 `presentedCategory` 作为当前资料分类页面的唯一呈现状态。
- 移除 `categoryDetailWasPresented` 和 `closeMenuAfterCategoryDetail()`。
- 移除分类 cover 的关闭后联动逻辑。
- 子页面返回只将 `presentedCategory` 置空。
- 更多菜单右上角关闭按钮仍调用 `onClose()`，返回 `XAgeMainView`。

导航结果为：

`XAgeMainView → XAgeMoreMenu → 分类子页面 → XAgeMoreMenu`

### 账号与安全

在账号管理区域新增“账号与安全”入口，通过独立全屏页面展示：

1. 手机号
   - 页面出现时请求 `/api/users/me`。
   - 对标准 11 位手机号仅展示前三位和后四位，例如 `138****1234`。
   - 若手机号为空或格式不符合预期，展示“暂未获取”，不展示原始异常值。
   - 此行仅用于展示，不响应点击。
2. 修改密码
   - 点击后以 sheet 展示现有 `ChangePasswordSheet`。
   - 表单包含旧密码、新密码、确认密码和保存操作。
   - 继续使用既有校验：旧密码必填、新密码至少 8 位、两次输入一致、新旧密码不同。
   - 保存成功自动关闭修改密码页；失败保留表单并显示错误。
3. 注销账号
   - 使用危险操作样式。
   - 继续复用 `XAgeDeleteAccountSheet`，要求输入“注销”后才允许提交。
   - 请求进行中禁止重复提交和手势关闭。
   - 仅在服务端删除成功后关闭更多菜单并清除当前登录态。

注销入口仅保留在“账号与安全”页面；账号管理首页不重复展示独立注销行。退出登录仍保留在账号管理首页。

### 隐私政策

- 在关于区域增加“隐私政策”入口。
- 使用独立全屏 SwiftUI 页面，不依赖网络。
- 将 `docs/privacy.html` 的标题、更新时间和八个章节转换为本地结构化文本。
- 使用可滚动布局和 XAGE 既有背景、卡片、字体颜色与返回按钮风格。
- 页面返回只关闭隐私政策页面并回到更多菜单。

### 权限申请与使用情况说明

- 在关于区域增加“权限申请与使用情况说明”入口。
- 使用独立全屏 SwiftUI 页面。
- 根据当前工程声明展示相机、相册读取、相册写入、麦克风、语音识别、Apple 健康读取和 Apple 健康写入说明。
- 每项包含权限名称、申请时机、使用目的以及拒绝后的影响。
- 明确权限由用户决定，可在 iOS 系统设置中调整；未授权项目不会读取。
- 页面返回只关闭权限说明页面并回到更多菜单。

## 状态与数据流

`XAgeMoreMenu` 新增账号安全、隐私政策和权限说明三个布尔呈现状态。每个全屏页面仅持有自身局部交互状态，关闭时只修改对应状态，不调用更多菜单的 `onClose()`。

账号安全页负责获取用户信息，但账号删除仍使用更多菜单现有的账号 ViewModel 和登录态清理逻辑，以确保服务端成功、弹层关闭和本地登出之间保持原有顺序。修改密码继续由既有 `ChangePasswordViewModel` 独立管理。

## 错误与边界处理

- 用户信息请求失败时显示手机号占位内容和可理解的错误提示，页面其他操作仍可使用。
- 修改密码提交期间禁用保存按钮；错误不清空用户输入。
- 注销失败时不清除登录态、不关闭更多菜单，并展示服务端错误。
- 注销成功后使用操作开始时捕获的 token 清理对应登录态，避免异步过程中误清除新登录账号。
- 所有新增页面支持小屏滚动和安全区域，不依赖固定内容高度。
- 本地隐私与权限内容不因离线或服务端不可用而无法访问。

## 非目标

- 不修改后端账号、密码或注销接口。
- 不引入远程隐私政策网页或 WebView。
- 不重构整个 App 或更多菜单的全局导航架构。
- 不改变隐私政策原有法律表述；仅将现有内容转换为 App 内本地展示。

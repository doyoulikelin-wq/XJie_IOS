# XAGE 用药快捷添加与问题反馈设计

日期：2026-07-14

## 目标

本次改动包含两个相互独立但规模较小的 XAGE 界面增强：

1. 在用药新增和编辑表单中，为剂量、频次和使用说明提供常用内容的快捷气泡。
2. 在“更多 → 关于”中新增“问题反馈”入口，复用现有反馈接口提交用户的问题或建议，并展示官方联系邮箱。

## 范围

### 包含

- 新版 `XAgeMedicationManagementView` 的新增、编辑用药 Sheet。
- 新版 `XAgeMoreMenu` 的“关于”分组。
- 复用现有 `SettingsViewModel.submitFeedback()` 和 `/api/feedback`。
- 新增相关单元测试、UI 测试和构建验证。

### 不包含

- 不修改旧版 `SettingsView` 的反馈页面。
- 不修改用药服务端数据结构和接口。
- 不增加用户联系方式输入框。
- 不实现邮件发送或跳转系统邮件客户端。
- 不重构旧版和新版反馈页面为同一个共享 UI。

## 方案选择

采用 XAGE 专用反馈 Sheet，并复用现有反馈 ViewModel 和后端接口。快捷气泡使用用药页面现有的流式布局组件。

没有直接复用旧版 `FeedbackSheet`，因为该页面是私有的旧版 `NavigationView + Form` 风格；也不进行新旧页面统一重构，以避免扩大本次改动范围。

## 用药快捷添加

### 预设内容

- 剂量：`半片`、`1片`、`2片`、`5mg`、`10mg`
- 频次：`每日1次`、`每日2次`、`每日3次`、`睡前1次`、`按需服用`
- 使用说明：`饭后服用`、`随餐服用`、`空腹服用`、`睡前服用`、`整片吞服`

### 页面布局

在剂量、频次和使用说明输入框下方分别增加“快捷添加”标题与气泡按钮。气泡复用 `XAgeMedicationFlowLayout`，在较窄设备上自动换行，并保持现有 XAGE 液态玻璃视觉样式和至少 44 点的可点击区域。

### 点击行为

- 剂量气泡直接替换 `dosage` 当前值。
- 频次气泡直接替换 `frequency` 当前值。
- 使用说明气泡追加到 `instructions` 末尾：
  - 当前内容为空或只有空白时，直接写入快捷语。
  - 当前存在有效内容时，追加中文逗号 `，` 和快捷语。
  - 允许重复点击同一气泡，每次点击均按规则追加。

快捷操作只改变表单本地状态，用户仍需点击“保存”才会产生服务端请求。由于表单状态发生变化，现有的未保存修改检测和放弃确认逻辑继续生效。

使用说明的拼接规则提取为可独立测试的纯函数，避免把字符串边界逻辑散落在 SwiftUI 按钮闭包中。

## 问题反馈

### 入口与展示

在 `XAgeMoreMenu` 的“关于”分组新增一张 `XAgeAccountMenuRow`：

- 标题：问题反馈
- 图标：对话气泡类 SF Symbol
- 副标题：提交 APP 问题或改进建议

点击后以大尺寸 Sheet 打开 XAGE 专用反馈页面。页面包含：

- 标题和关闭按钮。
- 一个多行反馈输入框。
- 当前字数与 2000 字上限提示。
- 联系邮箱 `jianjieaitech@163.com` 的展示卡片。
- 提交按钮。

### 校验与提交

- 提交前清理内容首尾空白。
- 有效内容长度为 2～2000 字。
- 调用 `SettingsViewModel.submitFeedback()`。
- 提交参数使用固定分类 `general`，`contact` 传 `nil`。
- 复用现有接口 `/api/feedback`，由 ViewModel 自动附加 iOS 平台和 APP 版本。

### 状态与错误处理

- 提交期间禁用关闭和重复提交，并显示提交中状态。
- 提交成功后关闭反馈 Sheet，并由“更多”页面显示“反馈已提交”提示。
- 提交失败时保留输入内容，页面不关闭，并展示 ViewModel 返回的错误信息。
- 用户未输入内容或内容不满足长度限制时，提交按钮不可用。

## 状态归属

- `XAgeMoreMenu` 持有反馈 Sheet 是否展示及提交成功提示状态。
- XAGE 反馈 Sheet 自己持有输入内容与提交中状态。
- `SettingsViewModel` 继续负责网络提交和错误信息。
- 用药快捷内容继续由 `XAgeMedicationEditSheet` 的本地 `@State` 持有，不增加新的引用类型 ViewModel。

## 可访问性

- 三组快捷气泡具有可识别的按钮标题和稳定的 accessibility identifier。
- 反馈入口、输入框、邮箱展示和提交按钮增加稳定的 accessibility identifier，供 UI 测试定位。
- 所有气泡与操作按钮维持至少 44 点点击区域。

## 测试策略

按测试驱动顺序实现。

### 单元测试

- 空字符串追加快捷说明时直接返回快捷语。
- 纯空白字符串追加时不产生前置逗号。
- 已有内容追加时使用中文逗号连接。
- 剂量和频次快捷操作的目标行为为替换当前输入。

### UI 测试

- “更多 → 关于”存在“问题反馈”入口。
- 反馈 Sheet 能打开，并显示反馈输入框、联系邮箱和提交按钮。
- 新增用药页面能看到剂量、频次和使用说明三组快捷气泡。
- 点击剂量和频次气泡后，对应输入框显示预设值。
- 点击使用说明气泡后，输入框按追加规则更新。

### 构建验证

- 运行相关单元测试。
- 运行新增或扩展的 XAGE UI 测试。
- 执行 Debug 模拟器构建，确认没有 SwiftUI 类型检查超时或编译错误。
- 执行 `git diff --check`。

## 修改边界

用户允许清除 `struct XAgeMedicationManagementView : View` 中多余的冒号前空格。除此之外，已有未提交文件和与本需求无关的改动均不覆盖、不暂存。

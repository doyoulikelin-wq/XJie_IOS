# Simulator 逐步人工验证日志

## 环境

- Simulator：iPhone 17 Pro，iOS 26.3.1。
- App：iOS `XAGE` Debug 构建。
- 后端：本机 `127.0.0.1:18765`、独立 SQLite 测试库、合成用户和显式 AI 授权。
- 操作：通过 Computer Use 控制 Simulator，坐标点击仅用于 App 画面未暴露 AX 元素的情况。
- 数据：全部为合成测试输入；日志、文档和截图不保存 token、密码或 API key。

## 执行过程

### 1. 登录、数据源和授权

1. 启动 App，确认新 XAGE 数据页可进入且 API 指向本地服务。
2. 切换问答，询问 Apple 健康同步状态，确认回答使用数据库记忆，不输出 `apple_health: fresh` 等内部字段。
3. 把 AI 授权设为关闭，发送消息，确认出现授权弹层。
4. 点击拒绝，确认原消息仍在且不生成助手回答。
5. 再次发送并接受，确认同一消息重试且只出现一个用户气泡。

### 2. 路由、证据和主体

1. 发送 HRV 深度分析，观察服务器路由和进度步骤。
2. 在样本不足时确认系统不声称“下降趋势”，而是指出缺少时间窗和样本。
3. 询问家属同步状态，确认不使用本人 Apple 健康/CGM 代答。
4. 连续询问母亲、妻子、朋友病例，再显式切回本人，确认主体和可用数据一起切换。
5. 在上轮血压后改问血糖 5.5，发现历史主题污染；修复当前概念优先后重测通过。

### 3. 高风险数值与急症

1. 输入 `SBP 190`，确认单项严重收缩压进入确定性高风险路径。
2. 输入 `SBP 120`，确认系统询问缺失舒张压，不构造组合血压。
3. 输入血糖 20 无单位，确认同时解释 mmol/L 和 mg/dL 风险并要求立即核对。
4. 输入血糖 50 mg/dL，确认严重低血糖处理和不能吞咽边界。
5. 输入高血糖伴呕吐但否认腹痛，首次回答曾附带未提供的腹痛/深快呼吸；改为结构化保存实际 active symptoms 后重测只出现呕吐。

### 4. 网络、重试和会话隔离

1. 关闭本地后端后发送消息，发现前台请求可等待约 90 秒；把 URLSession 改为不等待连接并给出可重试状态。
2. 恢复服务并点击原气泡重试，确认服务器只保存一组 user/assistant 消息。
3. 在旧请求进行中创建新对话，确认新页面立即为空。
4. 等待旧请求完成，确认旧回答没有进入新对话。
5. 通过数据库租约测试两个会话争抢同一 `client_message_id`，确认只有新租约持有者可保存回答。

### 5. 孕产和儿童特殊人群

1. 输入“我老婆怀孕 32 周，现在血压 160/110，没有头痛”。
2. 确认首屏使用孕产 `160/110` 阈值、产科急诊、15 分钟内复测和 120 边界，不出现普通成人 `180/120`。
3. 下一轮输入“她现在血压 165/112，没有头痛”，确认“她”延续到妻子孕期主体。
4. 输入“说回我，我现在血压 190/125，没有胸痛”，确认恢复普通成人 `180/120` 规则且无产科文案。
5. 新对话输入“我老婆孕 34 周，血压 165/112，现在剧烈头痛，但没有上腹痛”，确认急症回答只写剧烈头痛，不写上腹痛。
6. 输入“5 岁孩子血糖 2.8 mmol/L”，确认不固定套用成人 15 克并要求 15 分钟复测。
7. 新对话输入“我血糖 2.8 mmol/L”，确认成年人仍使用 15 克/15 分钟规则。
8. 先发送妻子怀孕 8 周，再发送“我老婆确认没怀孕，她现在血压 190/125”，确认最新明确状态覆盖旧孕期记录，回复转为普通成人规则。

### 6. 中文输入法残留缺陷

1. 使用 Simulator 简体拼音键盘逐段输入“5 岁孩子血糖 2.8 mmol/L”。
2. 点击候选词后点击发送，消息和回答均正常，但输入框残留原文，存在再次点击发送造成重复请求的风险。
3. 修复为：发送按钮同步消费并清空草稿、退出输入焦点、下一主线程周期清理 IME 对同一已发送文本的回写。
4. 增加保护：若用户已经输入不同的新草稿，不执行二次清理。
5. 重新构建并按相同步骤输入、选择候选、发送；输入框立即显示占位文案，消息仅出现一次。证据见截图 30。

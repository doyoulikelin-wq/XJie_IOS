# iOS XAGE 报告模块当前运行 UX / 可访问性审计

日期：2026-07-15

审计类型：组合审计（UX + 可访问性风险）

范围：`历史报告 → 报告字段复核 → 已确认/评分待更新 → 本次报告解读与原始报告`。本报告只使用本轮运行导出的 4 张截图及对应的确定性 UI 自动化结果。

## 结论

**部分通过，但尚未完成视觉验收，也不构成发布就绪证明。**

报告历史、字段复核、报告级确认和“评分待更新”状态的层级清楚；日期、医院、资料类型、原始值、候选值、参考范围、冲突原因和原件位置都有可见文字，用户不需要只靠颜色判断状态。专项自动化路径精确通过 `1/1`。

阻断项出现在“原始报告”预览：当前证据中预览区域是一整块大面积纯黑矩形。它来自确定性黑色/无效夹具，无法证明真实 PDF 或图片原件能够正确渲染，因此**原始文件视觉呈现不予验收**。必须换成内容已知、肉眼可核对的有效 PDF/图片夹具后重新截屏和复跑。

本轮没有导出健康画像和用药模块截图，所以这不是全 App 审计；也不能据此判断健康画像、用药、真机辅助功能或 TestFlight 候选是否可发布。

## 用户目标与检查路径

用户应能在不误解处理状态的前提下完成四件事：找到一份报告、逐项核对识别结果、明确确认整份报告、查看带来源追溯的解读和原始资料。

1. **进入历史报告 — 健康**
   从报告管理页进入历史报告，检查新报告和旧版报告是否清楚区分，并核对日期、医院和资料类型。
2. **打开字段复核并完成确认前检查 — 基本健康**
   打开 `2026年体检报告.pdf`，检查原始值、候选值、参考范围、冲突原因和原件位置；完成字段决策后再启用报告级确认。
3. **提交整份报告确认 — 健康**
   确认后检查状态是否明确变为“已确认 · 评分待更新”，且主动作切换为“查看本次解读”，避免把报告入库误写成评分已经完成。
4. **查看本次解读和原始报告 — 阻断视觉验收**
   检查确认记录、不可变确认事件、来源位置和原始报告；文本追溯可见，但原始预览为纯黑夹具，无法验收真实文件渲染。

## 已确认的优点

- 可信状态有清晰的文字分层：待确认、历史未验证、已确认、评分待更新没有混用。
- 关键医疗数据同时呈现原始值、候选值、单位、参考范围、置信度、冲突解释和原件位置，减少“AI 为什么这样判断”的黑箱感。
- 报告级确认是独立、显式动作，且说明确认范围；不会因为单个字段通过就静默把整份报告写入可信数据。
- 页面主动作会随状态变化，从“确认整份报告并入库”切换为“查看本次解读”，动作与系统状态一致。
- 返回、关闭、开关和主按钮在截图中都有明显视觉边界；实际命中区尺寸仍需运行时测量。

## 可见问题与修复优先级

1. **阻断验收：原始文件预览为大面积纯黑。** 以内容已知的有效多页 PDF、JPEG/HEIC 和不可解码文件分别建夹具；成功态必须能核对页内已知文字/图形，失败态必须显示明确原因、重试和“打开原件”替代入口，而不是黑块。
2. **中优先级：不可变事件暴露过多小数位。** 用指标定义的显示精度格式化用户界面，避免 `8.20000000` 造成“数据异常或系统未完成”的错觉。
3. **证据缺口：固定标题与长内容边界未覆盖。** 补充顶部、底部、超长字段、超长医院/文件名、小屏和大字号截图，验证内容不会被固定标题或底部主按钮遮挡。
4. **范围缺口：健康画像与用药没有当前运行截图。** 必须单独导出完整路径和关键状态后才能把本报告扩展为全 App UI/交互审计。

## 可访问性风险与设备限制

截图可以确认状态没有只靠颜色表达，但不能证明 VoiceOver、动态字体、焦点顺序或真实触控行为。仍需完成以下检查：

- 在真实 iPhone 上用 VoiceOver 核对历史卡片、返回/关闭、状态标签、字段确认、报告级开关、主按钮、来源信息和原始报告的朗读顺序、名称、值、提示与激活动作。
- 使用最大辅助功能字号验证固定标题、底部按钮、长文件名、冲突说明和来源行是否重排、截断或互相遮挡；同时覆盖 iPhone SE 小屏。
- 实测半透明卡片上的浅蓝灰次级文字与背景对比度；截图观感不能替代颜色取样和对比度计算。
- 给原始报告预览提供可理解的辅助功能标签、页码、加载/失败状态和可替代打开方式；纯黑预览对视觉用户和辅助技术用户都没有可核对信息。
- 真机验证 PDF/多页图片缩放、翻页、旋转、内存压力、损坏文件恢复和系统“打开原件”行为。Simulator 的确定性夹具不能替代这些边界。

## 重新验收条件

报告模块至少满足以下条件后，才能把当前“部分通过”改为“视觉验收通过”：

1. 有效、内容已知的 PDF/图片原件在解读页可见，并能与已知页码/内容人工核对。
2. 损坏或不支持文件显示明确失败状态、原因和恢复入口，不出现无解释黑块。
3. 顶部/底部边界、长内容、小屏和最大辅助功能字号有当前运行截图。
4. VoiceOver 与真实设备文件预览完成可重复检查并留下证据。
5. 健康画像和用药模块另行完成当前运行截图审计；在此之前不得称为“全 App 审计完成”。
